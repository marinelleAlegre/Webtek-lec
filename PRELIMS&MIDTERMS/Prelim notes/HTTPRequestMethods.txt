HTTP Request Methods



GET
- It is used to request from the server about the retrieval 
of the source being identified by the request URl and it is 
returned as an entity in the message body



HEAD
- Identical to GET, except the message body is not included 
in the response 



POST
- Request that the origin server accept the 
entity enclosed in the request as a new subordinate of 
the resource identified by the Request-URI in the Request-Line. 



OPTIONS
- Request for information about the communication options
 available on the request/response chain identified by the Request-URI.

 

TRACE
- Request the server to “echo” back to the client the received 
request 



PUT
- Request the server to store the enclosed entity in the message 
under the specified request URI 



DELETE 
- Request the server to delete the resource identified by 
the request URI 



CONNECT
- Reserved for use of tunneling proxy servers
